The Console

In the beginning

  • was the command line
  • A text-only mechanism for input and output.
  • It can be used to control systems.
  • We use it as an environment for our simple programs.

But

The command line, i.e console, is very restricted.

  • We only use the console for text output and text input.
  • There is no nice graphical interation.
  • You need to learn to interact in computers using a less forgiving metaphor.

Output only

Output to the console prints a single character at a time.

Everything in the console is a character.

The end of line is marked with an endl character.

  • No endl no end of line.

No Backsies

  • After outputting a character, you can't go back and change it.
  • You can't clear the screen

any designs you have must take this into account i.e. don't try and do something impossible.

Purely C++11

std::cout

  • The C++ mechanism for outputting to screen.
  • std:: refers to the fact that it is in the C++ standard.
  • cout is short for console out.

std::endl

  • endl means end-of-line.
  • We need a special symbol for end of line as
    • UNIX uses \n.
    • Windows uses \r\n.
    • other platforms vary.

iostream

  • Both std::cout and std::endl live in iostream.
  • iostream is the C++ Input/Output stream library.
  • We need to include iostream into our programs.
#include <iostream>

<<

  • The << symbol (we'll call it a single symbol) directs output.
  • We can direct information to the console out.

Hello World

First program

#include <iostream>

int main() {
  std::cout << "Hello, world" << std::endl;
  return 0;
}

Notation

parentheses
( is a left parenthesis and ) is a right parenthesis. You might commonly call these brackets, but programmers use the technical names for them.
braces
we've already seen { and } referred to as braces or curly braces.
angular brackets
the < and > characters are left and right angular brackets respectively.
colons
the : is a colon, whereas ; is a semi-colon.

Anatomy

  • Included libraries at the top.
  • Some strange int main() stuff.
  • Lines of code between the { and }.
  • Each code statement ends with a ;.

main()

  • Is the starting point defined by Ken Thompson.
  • All executable C++ programs have a main() function.
  • The program must have the signature:
    • int main()
  • And the code is contained between braces.

return 0

  • I'll explain this soon.
  • Right now, all programs need to return 0; on their last line.

Self Evaluation

  • Right now we're looking at simple things.
  • We need to have good style in our code.

Great Style

#include <iostream>

int main() {
  std::cout << "Hello, world" << std::endl;
  return 0;
}
  • Great use of indentation.

Acceptable Style

#include <iostream>
int main() {
std::cout << "Hello, world" << std::endl;
  return 0;
}
  • Janky indentation.

Poor Style

#include<iostream>
int main() {std::cout<<"Hello, world"<<std::endl;
return 0;
}
  • No use of indentation

Poor Style

#include <iostream>
#include <cstdlib>

int main() {
  std::cout<<"Hello, world"<<std::endl;
  std::system("CLR");

  return 0;
}
  • Windows only -- here you're using the Windows command intepreter to run the CLR command.
