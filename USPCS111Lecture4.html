First Function

Let's write a function that takes two integers and returns their product as a result.

This function has the signature:

int mult(int x, int y)

mult

#include <iostream>

int mult(int x, int y) {
  int z = x * y;
  return z;
}

int main() {
  std::cout << mult(2, 21) << std::endl;
  return 0;
}

Using mult

#include <iostream>

int mult(int x, int y) {
  int z = x * y;
  return z;
}

int main() {
  int a(2);
  int b(21);
  int x = mult(a, b);
  std::cout << x << std::endl;
  return 0;
}

Testing Functions

Rule of 3

  • The rule of 3 asks you to pick:
    • 3 values you expect to work
    • 3 values you expect to not work
  • Test that the function works for the former and fails for the latter!

mult again

  • I expect the following pairs to work for mult;
    • (0,0),
    • (-1,-1), and
    • (2, 2).
  • The following pairs won't work as expected:
    • (2^32, 2^32)
    • (-2^32, -2^32)
    • (1, 2^64)

assert

Returning void

  • Some functions don't return anything.
  • These should be rare.
  • If your function returns nothing, you may wish to consider why it is there.
    • c.f. side-effects only.
  • Returning integers is an int return, returning nothing is void.

Print Menu

You might want to print a price list menu such as:

void print_menu() {
  std::cout << "1.   70c Super Ice Cream" << std::endl;
  std::cout << "2. $1.40 SuperAwesome Ice Cream" << std::endl;
  std::cout << std::endl << "Choose either 1 or 2 :" << std::endl;
}

But Qestion!

If I want a menu, surely I want the user's choice back. So

void print_menu()

should be

int print_menu()

or, even better,

enum class UserChoice {SUPER, SUPER_AWESOME};
UserChoice print_menu()

Problem Statement
